# 部位训练页面跳转与组件语义说明

## 1. 这份文档解决什么问题

这份文档专门说明 `pages/partTraining` 目录下：

- 哪些文件是“页面”
- 哪些文件只是“页面里的内容组件”
- 用户做了什么操作，会跳到哪里
- 整条训练链路在产品上是怎么串起来的

重点回答两个容易混淆的问题：

- `pages/partTraining/course-detail-adapter.vue` 到底是什么
- `pages/partTraining/components/course-detail-content.vue` 到底对应用户看到的哪一块

---

## 2. 先区分：页面 和 组件

### 2.1 真正的页面

这些文件是 `pages.json` 注册过的页面，属于真实路由页面：

- `pages/partTraining/part-training.vue`
- `pages/partTraining/course-detail-adapter.vue`
- `pages/partTraining/warm-up-page.vue`
- `pages/partTraining/adjust-equipment.vue`
- `pages/partTraining/formal-training.vue`
- `pages/partTraining/cool-down-page.vue`

这些页面会进入 uni-app 的页面栈。

### 2.2 不是页面，只是组件

这些文件不是独立路由，它们只是被页面或弹窗引用的组件：

- `pages/partTraining/components/course-detail-content.vue`
- `components/modals/ai-recommend-modal.vue`
- `components/modals/ai-recommend-result-modal.vue`

这些不会单独进入页面栈。

也就是说：

- 页面 = 真跳转
- 组件 = 真显示，但不单独算一页

---

## 3. `course-detail-adapter.vue` 和 `course-detail-content.vue` 的关系

### 3.1 `course-detail-adapter.vue` 是“详情页外壳”

文件：

- `pages/partTraining/course-detail-adapter.vue`

产品语义：

- 它对应的是“点击课程卡片后打开的那一个完整详情页”
- 这个文件自己不负责画复杂 UI
- 它只负责做两件事：
  - 从路由参数里拿到课程 `id`
  - 根据 `id` 找课程数据
  - 再把这门课的数据传给 `course-detail-content.vue`

你可以把它理解成：

- 页面壳
- 数据转接层
- 路由入口页

### 3.2 `course-detail-content.vue` 是“详情页主体内容”

文件：

- `pages/partTraining/components/course-detail-content.vue`

产品语义：

- 它对应用户肉眼看到的“课程详情主内容”
- 包括：
  - 顶部标题区
  - 封面图
  - 标签和时长
  - 课程简介浮卡
  - 注意事项
  - 动作预览
  - 底部“开始训练”按钮

所以两者关系可以直接理解成：

- `course-detail-adapter.vue` = 整个详情页这张“页面容器”
- `course-detail-content.vue` = 容器里面真正被用户看到的大块内容

### 3.3 为什么要拆成这两层

因为 `course-detail-content.vue` 这块内容，后来不只详情页要用，AI 推荐结果弹窗也要用。

所以现在的复用关系是：

- 独立详情页里复用它
- AI 推荐结果弹窗里也复用它

这样不用维护两套课程详情 UI。

## 4. 页面与组件对应表

### 4.1 课程库首页

文件：

- `pages/partTraining/part-training.vue`

- AI 推荐入口

文件：

- `pages/partTraining/course-detail-adapter.vue`

- `pages/partTraining/components/course-detail-content.vue`

用户看到的是：

- 从课程库点进去后的完整课程详情页

- `pages/partTraining/warm-up-page.vue`

### 4.4 调整器械页

文件：

- `pages/partTraining/adjust-equipment.vue`

用户看到的是：

- 器械调整说明
- 示例卡片
- 开始训练按钮

### 4.5 正式训练页

文件：

- `pages/partTraining/formal-training.vue`

用户看到的是：

- 正式训练过程页
- 训练状态信息
- 上一步 / 下一步 / 退出

### 4.6 拉伸收操页

文件：

- `pages/partTraining/cool-down-page.vue`

- 这是弹窗组件，不是页面

文件：

- `components/modals/ai-recommend-result-modal.vue`

内部主要复用：

- `pages/partTraining/components/course-detail-content.vue`

用户看到的是：

- 推荐课程详情弹窗

注意：

- 这也是弹窗组件，不是页面
- 不进入页面栈

## 5. 按用户操作梳理页面链路

## 5.1 普通选课训练链路

### 步骤 1：进入部位训练首页

页面：

- `pages/partTraining/part-training.vue`

用户动作：

- 打开部位训练页面

### 步骤 2：点击右侧课程卡片

触发位置：

- `pages/partTraining/part-training.vue`
- `handleCourseClick()`

跳转到：

- `pages/partTraining/course-detail-adapter.vue?id=课程id`

### 步骤 3：进入课程详情页

页面：

- `pages/partTraining/course-detail-adapter.vue`

内部渲染：

- `pages/partTraining/components/course-detail-content.vue`

用户看到的是：

- 一整页课程详情

### 步骤 4：点击“开始训练”

触发位置：

- `pages/partTraining/components/course-detail-content.vue`
- `startTraining()`

跳转到：

- `pages/partTraining/warm-up-page.vue?courseId=课程id`

### 步骤 5：热身完成或跳过热身

页面：

- `pages/partTraining/warm-up-page.vue`

用户动作：

- 等待热身进度自然完成
- 或点击“跳过热身”
- 或控制面板里点“下一训练环节”

跳转到：

- `pages/partTraining/adjust-equipment.vue`

### 步骤 6：点击“开始训练”

页面：

- `pages/partTraining/adjust-equipment.vue`

用户动作：

- 点击页面底部“开始训练”

跳转到：

- `pages/partTraining/formal-training.vue`

### 步骤 7：正式训练进入拉伸

页面：

- `pages/partTraining/formal-training.vue`

用户动作：

- 点击“下一训练环节”

跳转到：

- `pages/partTraining/cool-down-page.vue`

### 步骤 8：拉伸结束，完成训练

页面：

- `pages/partTraining/cool-down-page.vue`

用户动作：

- 拉伸进度完成
- 或点击“跳过拉伸”

结果：

- 先显示“训练完成弹窗”
- 关闭完成弹窗后回到：
  - `pages/partTraining/part-training.vue`

---

## 5.2 AI 推荐训练链路

### 步骤 1：在部位训练首页点击 AI 推荐

页面：

- `pages/partTraining/part-training.vue`

用户动作：

- 点击左侧 `AI推荐`

结果：

- 不跳页面
- 打开弹窗组件：
  - `components/modals/ai-recommend-modal.vue`

### 步骤 2：填写基础信息并点击“开始分析”

组件：

- `components/modals/ai-recommend-modal.vue`

结果：

- 不跳页面
- 只是把表单数据抛给 `part-training.vue`

### 步骤 3：页面层计算推荐结果

页面：

- `pages/partTraining/part-training.vue`

实际逻辑：

- `getAiRecommendationScore(form, course)`
- `getRecommendedCourse(form)`

注意：

- 这一步不是页面跳转
- 只是页面内计算“推荐哪门课”

### 步骤 4：弹出推荐结果

组件：

- `components/modals/ai-recommend-result-modal.vue`

内部复用：

- `pages/partTraining/components/course-detail-content.vue`

结果：

- 仍然不跳页面
- 只是从“基础信息弹窗”切换成“推荐结果弹窗”

### 步骤 5：在推荐结果弹窗中点击“开始训练”

组件：

- `components/modals/ai-recommend-result-modal.vue`

内部触发：

- 复用的 `course-detail-content.vue` 抛出开始训练事件
- 再由 `part-training.vue` 统一接管

最终跳转到：

- `pages/partTraining/warm-up-page.vue?courseId=课程id`

也就是说：

- AI 推荐链路不会经过独立的课程详情页
- 它会直接从推荐结果弹窗进入热身页

## 6. 链式页面栈示意

## 6.1 普通选课链

`part-training.vue`
-> `course-detail-adapter.vue`
-> `warm-up-page.vue`
-> `adjust-equipment.vue`
-> `formal-training.vue`
-> `cool-down-page.vue`
-> 训练完成弹窗
-> `part-training.vue`

说明：

- `course-detail-content.vue` 是详情页内容组件，不单独算一页

## 6.2 AI 推荐链

`part-training.vue`
-> `ai-recommend-modal.vue`（弹窗，不进页面栈）
-> `ai-recommend-result-modal.vue`（弹窗，不进页面栈）
-> `warm-up-page.vue`
-> `adjust-equipment.vue`
-> `formal-training.vue`
-> `cool-down-page.vue`
-> 训练完成弹窗
-> `part-training.vue`

说明：

- 两个 AI 弹窗都只是覆盖在 `part-training.vue` 上
- 它们不是真页面

---

## 7. 一个容易误解的点

### `adjust-equipment.vue` 还在主链路里，但不是所有“返回”都会回到它

原因：

- `warm-up-page.vue` 完成后，确实会进入 `adjust-equipment.vue`
- 但 `formal-training.vue` 的“上一训练环节”现在是回 `warm-up-page.vue`
- 不是回 `adjust-equipment.vue`

所以从产品体验上看：

- `adjust-equipment.vue` 更像一个“热身后、正式训练前的过渡页”
- 而不是一条在训练过程里来回穿梭的强节点

---

## 8. 当前最简理解

如果只记最重要的两句，可以记这个：

- `course-detail-adapter.vue` 是“课程详情页路由壳子”
- `course-detail-content.vue` 是“课程详情真正被用户看到的主体内容”

如果只记主链路，可以记这个：

- 普通选课：课程库 -> 详情页 -> 热身 -> 调器械 -> 正式训练 -> 拉伸 -> 完成
- AI 推荐：课程库 -> AI 基础信息弹窗 -> AI 推荐结果弹窗 -> 热身 -> 调器械 -> 正式训练 -> 拉伸 -> 完成
