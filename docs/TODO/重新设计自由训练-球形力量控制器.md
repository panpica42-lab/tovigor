## 一、组件结构

球形力量控制器位于 `pages/freeTraining/free-training.vue`，由以下层次结构组成：

```
.control-center                    # 控制区容器（flex居中，flex:1撑满剩余空间）
└── .dial-wrapper                  # 旋钮包装器（400rpx × 400rpx，position: relative）
    ├── .dial-arc.dial-arc-bg      # 圆弧背景（灰色底，始终显示）
    ├── .dial-arc-progress-wrapper # 圆弧进度容器（仅开机显示）
    │   └── .dial-arc.dial-arc-active  # 彩色进度圆弧（CSS mask 遮罩实现）
    ├── .dial-decoration           # 装饰光环（仅开机显示，底部平面效果）
    ├── .dial-circle               # 圆形中心球体（显示 kg 数值）
    └── .dial-power-btn            # 开关按钮（绝对定位于正中央）
        └── .power-icon            # 电源图标
```

### 各部分详解

| 组件 | 作用 | 定位方式 | z-index |
|------|------|----------|---------|
| `dial-wrapper` | 整个控制器的容器，绑定触摸事件 | relative | - |
| `dial-arc-bg` | 圆弧刻度背景，关机时变灰 | absolute | - |
| `dial-arc-progress-wrapper` | 进度遮罩容器，通过 CSS 变量控制角度 | absolute | - |
| `dial-arc-active` | 彩色进度圆弧，用 conic-gradient mask 裁剪 | - | - |
| `dial-decoration` | 底部光环装饰，营造 3D 平面效果 | absolute | 1 |
| `dial-circle` | 中央圆球，显示力量数值 | relative | 2 |
| `dial-power-btn` | 开关按钮，覆盖在圆球正中央 | absolute | 3 |

## 二、工作原理

### 2.1 触摸角度计算

用户在 `dial-wrapper` 区域触摸时，通过以下步骤计算力量值：

1. **获取圆心坐标**：页面渲染后（onReady）计算 `.dial-circle` 的中心点
2. **计算触摸向量**：`dx = touchX - centerX`, `dy = touchY - centerY`
3. **计算角度**：`atan2(dy, dx)` 转换为度数，再旋转 90° 使正上方为 0°
4. **限制范围**：角度限制在 [-120°, +120°]（圆弧范围）
5. **映射力量值**：线性映射到 [0, 100] kg

```javascript
// 角度 → 力量值映射公式
ratio = (angle - (-120)) / (120 - (-120))  // 归一化到 0~1
value = 0 + ratio * 100                     // 映射到 0~100 kg
```

### 2.2 开关状态

- **关机状态**：圆弧变灰、禁止触摸调节、力量归零
- **开机状态**：显示彩色进度、显示光环装饰、允许触摸调节

## 三、已发现的问题

### 🔴 问题1：点击开关时触发力量调节（严重）

**现象**：
- 点击开关**开启**时，出现意外的绿色填充或力量值跳变
- 点击开关**关闭**时，力量先飙升到接近 100kg，然后才关闭

**原因分析**：

`dial-power-btn` 使用了 `@tap.stop` 阻止 tap 事件冒泡，但父元素 `dial-wrapper` 绑定的是 `touchstart` 和 `touchmove` 事件。

**关键**：`@tap.stop` 只阻止 tap 事件，**不阻止 touchstart/touchmove**！

事件触发顺序：
```
用户点击开关按钮
    ↓
1. touchstart 事件冒泡到 dial-wrapper → 触发 onDialTouch()
2. tap 事件被 .stop 阻止冒泡 → 但 togglePower() 已执行
```

由于开关按钮位于正中央（top: 50%, left: 50%），触摸点相对于圆心的角度约为 90°（偏下方），映射后力量值约为：
```
angle ≈ 90° (或更大，取决于触摸点偏移)
ratio = (90 - (-120)) / 240 = 210/240 = 0.875
value = 87.5 kg → 四舍五入 88kg（或更高）
```

**解决方案**：

在 `dial-power-btn` 上添加 `@touchstart.stop` 和 `@touchmove.stop`，阻止触摸事件冒泡：

```vue
<!-- 修改前 -->
<view class="dial-power-btn" @tap.stop="togglePower">

<!-- 修改后 -->
<view class="dial-power-btn" 
    @tap.stop="togglePower"
    @touchstart.stop
    @touchmove.stop>
```

### 🟡 问题2：绿色填充效果需要确认（待观察）

**现象**：点击开启时出现"莫名其妙的绿色填充"

**可能原因**：
1. `dial-circle--on` 类添加了绿色渐变背景（这是设计意图）
2. `dial-arc-active` 的绿色 drop-shadow 可能过于明显
3. 进度条初始角度 -120° 时，mask 可能显示了不该显示的区域

**待确认**：修复问题1后，观察是否还有异常绿色填充。如果有，可能需要调整：
- 进度条 mask 的初始状态
- 开机动画的过渡效果
