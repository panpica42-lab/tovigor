# 串口模拟器（下位机离线模拟）

> 创建日期：2026-03-05  
> 状态：📋 待开发

---

## 需求背景

**痛点**：开发和调试 UI 界面时，必须连接真实下位机硬件才能测试通信相关功能，导致：
- 没有硬件时无法开发/演示
- H5 调试模式完全无法测试通信流程
- 每次调试都要连接设备，效率低

**目标**：实现一个模拟函数，在下位机离线时模拟回包，对 Vue 页面完全透明——启用模拟和真实通信没有任何区别。

---

## 解决方案

### 核心思路：在 serialService.js 层注入模拟

```
Vue 页面 --> serialService.js --> [分叉点]
                                    ├─ 真实模式 --> UTS 插件 --> 硬件
                                    └─ 模拟模式 --> 模拟器函数 --> 生成假 A9 帧
```

### 为什么选择这个位置？

1. **对 Vue 页面完全透明**：页面只调用 `startWorking()`、`on('frame', cb)`，不关心数据来源
2. **保持事件机制不变**：模拟器只需调用 `emit('frame', data)` 即可触发监听
3. **改动集中**：只需修改 `serialService.js` 一个文件
4. **一键切换**：通过 `SIMULATION_MODE` 开关控制真实/模拟模式

### 文件结构

```
utils/
├── serialService.js      # 主服务（添加模拟分支逻辑）
└── serialSimulator.js    # 【新建】模拟器（生成假 A9 帧数据）
```

### 需要改造的函数

| 函数 | 真实模式 | 模拟模式 |
|-----|---------|---------|
| `connect()` | 打开串口 | 直接返回成功，标记已连接 |
| `startWorking()` | 发送 D180 + 轮询读取 | 启动模拟定时器生成 A9 |
| `stopWorking()` | 停止定时器 | 停止模拟定时器 |
