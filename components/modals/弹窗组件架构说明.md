# 弹窗组件架构说明

## 🏗️ 组件层次结构

```
ModalContainerLight（最底层 - 极简容器）
  ↓
ModalBase（基础容器 - 遮罩+居中盒子）
  ↓
业务弹窗组件（ModalDialog、ConfirmModal 等）
```

---

## 📦 两个容器组件对比

### 1. ModalContainerLight（最底层、最轻量）

**文件：** `modal-container-light.vue`

**特点：**
- ✅ **极简设计**：只有遮罩层 + 插槽，无任何预设样式
- ✅ **完全自由**：不限制位置、动画、样式，全由你决定
- ✅ **体积最小**：代码量最少，性能最优
- ✅ **最底层**：其他弹窗容器都可以基于它构建

**提供的功能：**
- 遮罩层（无背景色，需自定义）
- 显示/隐藏控制（`v-model:visible`）
- 点击遮罩关闭（`closeOnClickOverlay`）
- `@overlay-click` 事件

**不提供的功能：**
- ❌ 无动画
- ❌ 无位置定位
- ❌ 无宽高圆角配置
- ❌ 无遮罩背景色（需自行设置）

**使用场景：**
- 需要完全自定义样式和动画的弹窗
- 性能要求极高的场景
- 构建其他弹窗容器的基础

**示例：**
```vue
<ModalContainerLight v-model:visible="show">
  <!-- 完全自定义：位置、样式、动画全靠你自己写 -->
  <view class="my-custom-modal" @click.stop>
    自定义内容
  </view>
</ModalContainerLight>

<style>
.my-custom-modal {
  position: absolute;
  top: 50%;
  left: 50%;
  transform: translate(-50%, -50%);
  background: white;
  /* 你的自定义样式... */
}
</style>
```

---

### 2. ModalBase（基础容器）

**文件：** `modal-base.vue`

**特点：**
- ✅ **简洁设计**：遮罩层 + 居中弹窗盒子
- ✅ **灵活配置**：宽度、圆角、背景色可调
- ✅ **淡入动画**：内置淡入淡出效果
- ✅ **纯容器**：通过插槽完全自定义内容

**提供的功能：**
- 遮罩层（半透明黑色背景）
- 居中定位的弹窗盒子
- 淡入淡出动画
- 样式配置（width、borderRadius、backgroundColor）
- 点击遮罩关闭

**使用场景：**
- 需要自定义内容结构的弹窗
- 作为业务弹窗的基础容器
- 灵活度要求较高的场景

**示例：**
```vue
<ModalBase 
  v-model:visible="show"
  :width="600"
  :border-radius="32"
>
  <view class="content">
    完全自定义的内容
  </view>
</ModalBase>
```

## 🎯 如何选择？

| 场景 | 推荐组件 | 原因 |
|------|---------|------|
| 标准对话框（确认/提示） | ✅ ModalDialog | 开箱即用，有标题+按钮 |
| 自定义内容弹窗 | ✅ ModalBase | 纯容器，内容完全自定义 |
| 需要特殊动画效果 | ✅ ModalContainerLight | 完全自定义 |
| 需要不规则布局 | ✅ ModalContainerLight | 不受位置限制 |
| 构建弹窗组件库 | ✅ ModalContainerLight | 作为基础层 |
| 快速原型开发 | ✅ ModalBase | 减少样式代码 |

## 📋 实际使用建议

### 推荐做法：根据场景选择

**标准对话框（确认、提示）用 ModalDialog：**
```vue
<ModalDialog 
  v-model:show="show" 
  title="提示"
  @confirm="handleConfirm"
>
  <text>确定要删除吗？</text>
</ModalDialog>
```

**自定义内容用 ModalBase：**
```vue
<ModalBase v-model:visible="show" :width="600">
  <view class="my-content">
    <!-- 完全自定义的内容 -->
  </view>
</ModalBase>
```

### 仅在以下情况使用 ModalContainerLight：

1. **需要非标准动画**（如旋转、3D 效果等）
2. **需要不规则定位**（如跟随鼠标、自适应位置等）
3. **构建自己的弹窗容器**（基于 Light 封装自己的 Container）
4. **极端性能优化**（减少不必要的代码）

## 🔧 迁移指南

### 从 BaseModal 迁移到 ModalBase

**代码无需修改，只需改名：**
```vue
<!-- 旧代码 -->
<BaseModal v-model:visible="show">

<!-- 新代码（只改导入和组件名） -->
<ModalBase v-model:visible="show">
```

**导入语句：**
```javascript
// 旧
import BaseModal from '@/components/base-modal.vue'

// 新
import ModalBase from '@/components/modals/modal-base.vue'
```

## 🎉 总结

- **ModalContainerLight** = 最薄、最自由、最底层（你想怎么玩就怎么玩）
- **ModalContainer** = 配置化、标准化、开箱即用（推荐日常使用）
- **业务弹窗** = 基于容器封装的具体功能组件

**记住：先用 ModalContainer，不够用再考虑 ModalContainerLight！**
