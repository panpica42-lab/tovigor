# 屏保页 → 首页 跳转优化方案

> **更新日期**: 2026-03-13

## 问题描述

从待机屏保页（idle.vue）点击进入首页（index.vue）时，明显感觉到页面加载慢，画面先出现上半部分，下半部分还没刷新出来。

## 问题原因分析

1. **`reLaunch()` 开销大**：会销毁当前页面栈再创建新页面，有明显切换延迟
2. **首页图片资源多**：点击后才开始加载大量 SVG、PNG 图片
3. **首屏渲染压力**：首页组件较复杂，首次渲染需要时间

## 优化方案

### 1. 图片预加载（已实现）

在 idle.vue 视频播放期间，通过隐藏的 `<image>` 组件预先加载首页关键图片：

```vue
<!-- 预加载关键图片（不可见） -->
<view class="preload-images">
  <image v-for="img in preloadList" :key="img" :src="img" mode="aspectFit" />
</view>
```

**预加载的图片清单**：
| 图片路径 | 用途 |
|---------|------|
| `/static/icons/homeActivity/bg_free_training.svg` | 自由训练卡片背景 |
| `/static/icons/homeActivity/bg_body_training.svg` | 部位训练卡片背景 |
| `/static/icons/homeActivity/bg_smart_assess.svg` | 智能评估卡片背景 |
| `/static/icons/homeActivity/btn_usage_guide.svg` | 使用指南按钮 |
| `/static/icons/homeActivity/ic_control_*.svg` | 顶部控制面板6个图标 |
| `/static/icons/games/flappyBirdGame/ic_bird.png` | 游戏图标 |
| `/static/icons/partTrainingActivity/ic_1x.jpg` | 课程图片1 |
| `/static/icons/partTrainingActivity/ic_2x.jpg` | 课程图片2 |
| `/static/icons/partTrainingActivity/ic_4x.jpg` | 课程图片3 |

**样式处理**（隐藏但触发加载）：
```css
.preload-images {
  position: absolute;
  left: -9999rpx;
  top: -9999rpx;
  width: 1rpx;
  height: 1rpx;
  overflow: hidden;
  opacity: 0;
  pointer-events: none;
}
```

### 2. 使用 redirectTo 替代 reLaunch（已实现）

```javascript
// ❌ 原来的方式 - 销毁整个页面栈再重建
uni.reLaunch({ url: '/pages/index/index' })

// ✅ 优化后 - 直接替换当前页面，性能更好
uni.redirectTo({
  url: '/pages/index/index',
  fail: () => {
    // 降级方案
    uni.reLaunch({ url: '/pages/index/index' })
  }
})
```

**区别**：
| API | 行为 | 性能 |
|-----|------|------|
| `reLaunch` | 关闭所有页面，打开新页面 | 较慢 |
| `redirectTo` | 关闭当前页面，打开新页面 | 较快 |

### 3. 防重复点击（已实现）

```javascript
let isNavigating = false

const goHome = () => {
  if (isNavigating) return
  isNavigating = true
  
  uni.redirectTo({
    // ...
    complete: () => {
      setTimeout(() => {
        isNavigating = false
      }, 500)
    }
  })
}
```

### 4. 移除 touchstart 事件（已实现）

```vue
<!-- ❌ 原来 - 会产生 passive event listener 警告 -->
<view @click="goHome" @touchstart="goHome">

<!-- ✅ 优化后 - 只用 click，触摸设备也能正常响应 -->
<view @click="goHome">
```

## 尝试过但撤销的方案

### 首页延迟渲染（已撤销）

曾尝试用 `v-if="pageReady"` 延迟渲染课程推荐区域，但导致该区域完全不显示，已撤销。

```javascript
// ❌ 问题代码 - 导致课程推荐区域消失
const pageReady = ref(false)
setTimeout(() => { pageReady.value = true }, 50)
```

### uni.preloadPage（不支持）

`uni.preloadPage()` 对 Vue 页面不支持，只支持 nvue 页面：

```
[idle] 首页预加载失败: {"errMsg":"preloadPage:fail can not preload vue page"}
```

## 后续可优化方向

如果当前优化效果仍不理想，可以考虑：

1. **图片格式优化**：将 SVG 转为更小的 PNG/WebP
2. **骨架屏**：首页加载时先显示灰色占位块，再平滑过渡到真实内容
3. **页面过渡动画**：添加淡入动画掩盖加载过程
4. **首页拆分**：将课程推荐做成独立组件，异步加载
