# Flappy Bird 串口集成说明

> 最后更新：2026-03-12

## 概述

本游戏通过检测用户拉绳动作（sportCount 变化）来控制小鸟跳跃。

## 串口连接

| 环节 | 负责方 | 说明 |
|------|--------|------|
| 连接串口 | App.vue | 应用启动时调用 `connect()` |
| 检查状态 | 游戏页面 | `onMounted` 中调用 `getStatus()` |
| 断开连接 | App.vue | 应用退出时处理 |

**页面不负责连接/断开**，只检查状态并据此切换输入模式：
- 串口已连接 → `inputMode = 'serial'`
- 串口未连接 → `inputMode = 'simulate'`（点击模拟）

## 轮询时机

| 时机 | 动作 | API |
|------|------|-----|
| 点击"开始游戏" | 启动轮询 + 周期发送 | `startWorking(10, FORCE_MODE.CONST, 200)` |
| 游戏结束 | 关闭力量 + 停止轮询 | `sendOnce(5, FORCE_MODE.OFF)` + `stopWorking()` |
| 退出页面 | 同上 | 同上 |

**不是进入页面就开始轮询**，而是用户主动开始游戏才启动。

## 力量控制

### 开启力量

```javascript
startWorking(10, FORCE_MODE.CONST, 200)
```

| 参数 | 值 | 说明 |
|------|-----|------|
| force | 10 | 10kg 恒力 |
| forceMode | CONST (1) | 恒力模式 |
| sendInterval | 200 | 5Hz 发送频率（下位机响应式，发包频率决定回包频率） |

### 关闭力量

```javascript
sendOnce(5, FORCE_MODE.OFF)  // 模式0 + 力量5 = 正常关闭
stopWorking()                 // 停止轮询
```

⚠️ **重要**：关闭力量时 force 必须非零！
- `模式0 + 力量非0` = 正常关闭（绳子松开）
- `模式0 + 力量0` = **硬件级紧急停车**（慎用！）

## 跳跃触发逻辑

使用 A9 上行帧中的 `sportCount` 字段（左右手运动次数之和）：

```javascript
let lastSportCount = 0

const handleSerialFrame = (data) => {
  const currentSportCount = data.parsed.sportCount
  
  if (currentSportCount > lastSportCount) {
    jump()  // 触发跳跃
    lastSportCount = currentSportCount
  }
}
```

**为什么用 sportCount 而不是 instantForce？**

| 维度 | sportCount（次数） | instantForce（力量值） |
|------|-------------------|----------------------|
| 信号质量 | ✅ 下位机已判断，抗噪声 | ❌ 原始瞬时值，噪声大 |
| 触发明确性 | ✅ 变化即触发 | ❌ 需设计阈值+冷却 |
| 采样问题 | ✅ 累加值不会漏 | ❌ 可能漏掉峰值 |
| 用户体验 | ✅ 一次拉绳=一次跳跃 | ⚠️ 需要学习 |
